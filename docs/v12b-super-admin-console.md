# V12B：超级管理员后台

V12B 的学习主题是：超级管理员后台，也可以理解为 SaaS 产品的“平台总控台”。

V12A 已经让用户可以提交反馈，管理员可以查看最近反馈。V12B 往前走一步：当产品里有很多用户、很多组织、很多反馈时，平台方需要一个独立后台来查看整体情况。

这里的“超级管理员”不是组织拥有者。组织拥有者只能管理自己组织里的成员和组织词库；超级管理员站在整个平台视角，可以查看所有用户、所有组织、所有反馈和组织成员关系。

## 这一版做什么

- 新增独立后台页面 `admin.html`。
- 主页面只在超级管理员登录时显示“进入后台管理”入口。
- 后台页面显示平台总览：用户数量、组织数量、反馈数量、未处理反馈数量。
- 后台页面显示用户列表：邮箱、平台角色、创建时间、加入组织数量。
- 后台页面显示组织列表：组织名称、创建时间、拥有者、成员数量。
- 后台页面显示反馈列表：反馈内容、提交人、所属组织、状态、提交时间。

## 暂时不做什么

V12B 先做查看型后台，不做危险操作：

- 不删除用户。
- 不删除组织。
- 不修改用户角色。
- 不封禁账号。
- 不强制移除组织成员。

这些操作在真实产品里风险很高，需要二次确认、操作审计、权限分级和恢复方案。我们先学清楚“平台后台如何读取和展示全局信息”。

## 关键概念

- `super admin`：超级管理员，平台级管理员。
- `admin console`：后台管理页。
- `global view`：全局视角，可以看整个平台的数据。
- `organization owner`：组织拥有者，只管理自己组织。
- `read-only admin`：只读后台，只查看不修改。
- `RLS`：数据库行级安全，限制谁能读写哪些数据。
- `RPC`：数据库函数，前端通过函数拿到已经整理好的后台数据。

## 权限设计

V12B 沿用现有的 `profiles.role = 'admin'` 作为超级管理员判断。

原因：

- 现在项目里已经有 `admin / user` 平台角色。
- `admin` 已经用于公共词库发布和反馈查看。
- 继续使用它可以减少学习阶段的复杂度。

重要边界：

- `profiles.role = 'admin'` 是平台角色。
- `organization_memberships.role = 'owner'` 是组织角色。
- 组织 owner 不等于超级管理员。

## 页面设计

`index.html` 是普通产品页面，负责词典、组织空间、反馈提交。

`admin.html` 是后台页面，负责平台级查看：

1. 顶部显示当前后台身份和返回主页面入口。
2. 总览区域显示 4 个数字卡片。
3. 用户区域显示所有注册用户。
4. 组织区域显示所有组织及成员摘要。
5. 反馈区域显示所有反馈。

如果非管理员打开 `admin.html`：

- 如果未登录，提示先回主页面登录。
- 如果已登录但不是管理员，提示没有权限访问后台。
- 不展示任何全局数据。

## 数据设计

前端不直接拼多张表查询全局数据，而是通过 Supabase RPC 读取后台数据。

新增 SQL 文件：

```text
docs/v12b-super-admin-console-schema.sql
```

计划新增 4 个只读函数：

- `get_admin_overview()`：读取用户数、组织数、反馈数、未处理反馈数。
- `list_admin_users()`：读取用户列表和加入组织数量。
- `list_admin_organizations()`：读取组织列表、拥有者、成员数量。
- `list_admin_feedback_reports()`：读取反馈列表。

所有函数都必须检查 `public.is_admin()`。如果不是超级管理员，数据库层直接拒绝读取。

## 前端模块设计

新增文件：

- `admin.html`：后台页面结构。
- `scripts/admin.js`：后台页面交互和渲染。
- `scripts/adminData.js`：后台数据读取和格式转换。

复用文件：

- `scripts/supabaseClient.js`：连接 Supabase。
- `scripts/permissions.js`：读取当前用户 profile，判断平台角色。
- `style.css`：继续使用现有视觉风格，补充后台布局样式。

## 测试计划

自动测试：

- 后台总览数据格式转换。
- 用户列表数据格式转换。
- 组织列表数据格式转换。
- 反馈列表数据格式转换。
- 非管理员状态下不应尝试读取后台 RPC。

手动测试：

1. 普通用户打开 `admin.html`，应看到无权限提示。
2. 组织 owner 但不是平台 admin，打开 `admin.html`，也应看到无权限提示。
3. 平台 admin 打开 `admin.html`，应看到总览、用户、组织、反馈。
4. 从主页面点击“进入后台管理”，能打开后台页。
5. 从后台页点击“返回词典”，能回到主页面。

## 真实 SaaS 还差什么

生产级后台通常还需要：

- 搜索、分页和排序。
- 操作审计。
- 危险操作二次确认。
- 不同后台权限，比如客服、运营、技术管理员。
- 数据导出。
- 更专业的日志和错误监控。
- 对敏感信息做脱敏处理。

V12B 先建立最小可理解版本：超级管理员能安全地看见平台全局信息。
